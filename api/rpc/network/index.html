네트워크

RPC API를 사용하면 노드 및 밸리데이터에 대한 상태 정보를 쿼리할 수 있습니다.​

노드 상태​

  • 메서드: status
  • 매개변수: []

  • JavaScript

무엇이 잘못될 수 있나요?​

API 요청이 실패하면 RPC 서버는 제한된 수의 잘 정의된 오류 변형과 함께 구조화된 오류 응답을 반환하므로, 클라이언트 코드는 가능한 모든 오류 사례를 철저하게 처리할 수 있습니다. JSON-RPC 오류는 오류 응답을 구조화하기 위해 verror 규칙을 따릅니다.

{
    "error": {
        "name": <ERROR_TYPE>,
        "cause": {
            "info": {..},
            "name": <ERROR_CAUSE>
        },
        "code": -32000,
        "data": String,
        "message": "Server error",
    },
    "id": "dontcare",
    "jsonrpc": "2.0"
}

주의

위 구조의 code, data, 및 message 필드는 레거시 항목으로 간주되며, 향후 사용되지 않을 수 있습니다. 이에 의존하지 마세요.

다음은 status 메서드에 의해 반환될 수 있는 오류 변형의 전체 목록입니다.

ERROR_TYPE
error.name		ERROR_CAUSE
error.cause.name		이유해결책
INTERNAL_ERRORINTERNAL_ERROR노드 자체에 문제가 있거나 과부하가 걸렸습니다.
  • 나중에 다시 시도하세요.
  • 다른 노드에 요청을 보내세요.
  • error.cause.info에서 자세한 내용을 확인하세요.

네트워크 정보​

노드 네트워크 연결의 현재 상태(활성 피어, 전송된 데이터 등)를 반환합니다.

  • 메서드: network_info
  • 매개변수: none

예시:

  • JSON
  • HTTPie
{
  "jsonrpc": "2.0",
  "id": "dontcare",
  "method": "network_info",
  "params": []
}
http post https://rpc.testnet.near.org jsonrpc=2.0 method=network_info params:='[]' id=dontcare

검증 상태​

네트워크의 활성 밸리데이터에 쿼리하여 세부 정보 및 블록체인의 검증 상태를 반환합니다.

  • 메서드: validators
  • 매개변수: ["block hash"], [block number], 또는 최신 블록에 대해서는 [null]

참고: ["block hash"]와 [block number]의 경우, 에포크의 마지막 블록에서 쿼리해야 합니다.

[block number]

  • JSON
  • JavaScript
  • HTTPie
{
  "jsonrpc": "2.0",
  "id": "dontcare",
  "method": "validators",
  "params": [17791098]
}
const response = await near.connection.provider.validators(17791098);
http post https://rpc.testnet.near.org jsonrpc=2.0 method=validators params:='[17791098]' id=dontcare

  • HTTPie

무엇이 잘못될 수 있나요?​

API 요청이 실패하면 RPC 서버는 제한된 수의 잘 정의된 오류 변형과 함께 구조화된 오류 응답을 반환하므로, 클라이언트 코드는 가능한 모든 오류 사례를 철저하게 처리할 수 있습니다. JSON-RPC 오류는 오류 응답을 구조화하기 위해 verror 규칙을 따릅니다.

{
    "error": {
        "name": <ERROR_TYPE>,
        "cause": {
            "info": {..},
            "name": <ERROR_CAUSE>
        },
        "code": -32000,
        "data": String,
        "message": "Server error",
    },
    "id": "dontcare",
    "jsonrpc": "2.0"
}

주의

위 구조의 code, data, 및 message 필드는 레거시 항목으로 간주되며, 향후 사용되지 않을 수 있습니다. 이에 의존하지 마세요.

다음은 validators 메서드에 의해 반환될 수 있는 오류 변형의 전체 목록입니다.

ERROR_TYPE
error.name		ERROR_CAUSE
error.cause.name		이유해결책
HANDLER_ERRORUNKNOWN_EPOCH제공된 블록의 에포크를 데이터베이스에서 찾을 수 없습니다.
  • 요청한 블록이 올바른지 확인하세요.
  • 블록이 5 이상 에포크 전에 생성된 경우 아카이브 노드로 요청을 보내세요.
  • 요청된 블록이 일부 에포크의 마지막 블록인지 확인하세요.
REQUEST_VALIDATION_ERRORPARSE_ERROR전달된 인수는 JSON RPC 서버에서 구문 분석할 수 없습니다(인수 누락, 잘못된 형식 등).
  • 전달된 인수를 확인하고 올바른 인수를 전달하세요.
  • error.cause.info에서 자세한 내용을 확인하세요.
INTERNAL_ERRORINTERNAL_ERROR노드 자체에 문제가 있거나 과부하가 걸렸습니다.
  • 나중에 다시 시도하세요.
  • 다른 노드에 요청을 보내세요.
  • error.cause.info에서 자세한 내용을 확인하세요.
